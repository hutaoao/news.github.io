### Pretext：React核心成员开源高性能文本测量库

**摘要**：React核心团队成员Cheng Lou开源Pretext库，这是一个仅几KB大小的纯TypeScript文本测量库，UI渲染速度比传统DOM方案快约500倍，支持数十万个文本框丝滑滚动。

**补充**：Pretext通过避开DOM测量这一前端渲染瓶颈，采用自研文本测量逻辑，以浏览器字体引擎为基准进行精确测量。该库提供了两种使用场景：一是在不接触DOM的前提下测量文本高度，二是手动对段落进行分行排版。其创新架构天然适配AI迭代优化，为未来前端排版的智能化发展预留空间。目前已在GitHub获得2.82万星，开发者在X上展示了许多创意应用，如用文本制作游戏、MV和3D特效。

```typescript
// 示例：使用Pretext进行文本测量
import { prepare, layout } from 'pretext'

const handle = prepare("Hello, world!", { font: "16px Arial" })
const height = layout(handle, 300) // 在300px宽度下的高度
```

**来源链接**：[36Kr报道：Pretext开源项目](https://36kr.com/p/3747879330398981)

**影响评估**：高 - 对高性能文本渲染、富文本编辑器、聊天应用等场景有重大性能提升价值。

### HarmonyOS NEXT 7.0官宣6月发布，Mate 90首发

**摘要**：华为官方宣布纯血自研的HarmonyOS NEXT 7.0将于2026年6月在松山湖HDC开发者大会亮相，比原计划提前半年，Mate 90系列将全球首发预装该系统。

**补充**：HarmonyOS 7.0彻底移除安卓AOSP兼容层，实现1.1亿行代码全链路自研。主要改进包括：系统响应速度提升40%、后台保活率提升60%、内存占用降低30%、四年使用流畅度衰减率低于3%。该系统深度集成盘古大模型，小艺助手升级为主动智能体，支持本地AI计算保障隐私安全。目前鸿蒙全场景设备装机量已突破5000万，原生应用超35万款，覆盖日常99%使用场景。

### Chrome 146 Beta版发布，WebGPU能力持续增强

**摘要**：Chrome 146 Beta版本带来多项WebGPU更新，包括OpenGL ES 3.1和临时附件上的兼容模式支持，同时开发者工具新增自动连接和模拟MCP等功能。

**补充**：WebGPU在Chrome 146中进一步加强跨平台兼容性，为游戏开发、科学计算、机器学习等高性能应用提供更稳定的GPU加速基础。开发者工具方面，新增的MCP（Model Context Protocol）服务器功能让AI编码助手能够直接访问DevTools调试能力。此外，本次更新还优化了CSS边角形状实现、多列布局换行支持，并改进了Web推送通知的速率限制策略。

### ESLint性能挑战者Oxlint引发关注

**摘要**：前端社区热议Oxlint工具，作为ESLint的潜在替代方案，据称速度提升达100倍，引发开发者对现有代码检查工具性能优化的关注。

**补充**：Oxlint基于Rust开发，利用现代编译技术和并行处理大幅提升代码分析速度。虽然目前在规则覆盖和生态集成方面尚不如ESLint成熟，但其性能表现已引起大型项目团队的重视。特别是在持续集成环境中，代码检查耗时直接影响开发效率，高速lint工具的价值愈发凸显。社区讨论显示，Oxlint的成功也促使ESLint团队加速性能优化工作。

**来源链接**：[掘金技术社区讨论](https://juejin.cn/frontend)

**影响评估**：中 - 对大型项目构建优化有参考价值，值得关注后续发展。
